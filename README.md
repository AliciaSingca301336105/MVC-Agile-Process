# MVC Agile Process

## Agile Planning & Sprint Setup in Azure DevOps

### 📌 Step 1: Created an Epic
- **Epic:** Build and Enhance MVC Application

### 📌 Step 2: Created Features
- **Feature 1:** Create Basic MVC Application
- **Feature 2:** Enhance MVC with Search & Validation

### 📌 Step 3: Created User Stories & Tasks
#### Feature 1: Create Basic MVC Application
- **User Story 1:** "As a developer, I want to create an ASP.NET Core MVC project so that I can build a structured web application."
  - ✅ Task 1.1: Set up MVC project in Visual Studio
  - ✅ Task 1.2: Create Models, Views, and Controllers
  - ✅ Task 1.3: Implement Entity Framework and Database Connection
  - ✅ Task 1.4: Add CRUD operations for Movies table

#### Feature 1: Create Basic MVC Application
- **User Story 2:** "As a developer, I want to implement CRUD functionality sot that users can manage data."
  - ✅ Task 2.1: Create a simple user interface that allows users to perform Create, Read, Update, and Delete operations.
  - ✅ Task 2.2: As a developer, I want to implement CRUD functionality so that users can manage data.
 
#### Feature 2: Enhance MVC with Search & Validation
- **User Story 2:** "As a user, I want to search for movies so that I can find relevant data quickly."
  - ✅ Task 2.1: Implement search feature in Index view
  - ✅ Task 2.2: Add form validation using Data Annotations
  - ✅ Task 2.3: Test validation rules and error messages
 
#### Feature 2: Enhance MVC with Search & Validation
- **User Story 3:** "As a user, I want to search for movies so that I can find relevant data quickly."
  - ✅ Task 3.1: Add form validation using Data Annotations.
  - ✅ Task 3.2: Implement search feature in Index view.
  - ✅ Task 3.3: Test validation rules and error messages.

#### Feature 2: Enhance MVC with Search & Validation
- **User Story 4:** "As a user, I want validation on forms so that I don't submit incorrect data"
  - ✅ Task 4.1: Conduct thorough testing of the form validation to ensure errors are caught and provide appropriate feedback to users.
  - ✅ Task 4.2: Develop and apply validation rules for form inputs (e.g., required fields, format checks, length constraints, etc.).
